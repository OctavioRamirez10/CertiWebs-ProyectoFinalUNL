Modelo TCP/IP y Protocolos Clave

TCP/IP es el modelo práctico usado en Internet. Simplifica OSI en 4 capas. Los protocolos más importantes son:

  • IP (Internet Protocol) — Direccionamiento lógico y enrutamiento. IPv4 (32 bits) e IPv6 (128 bits).
  • TCP — Orientado a conexión, confiable, usa handshake de 3 vías (SYN, SYN-ACK, ACK).
  • UDP — Sin conexión, más rápido, sin garantía de entrega. Ideal para streaming y gaming.
  • HTTP / HTTPS — Transferencia de páginas web. HTTPS cifra con TLS.
  • DNS — Traduce nombres de dominio a direcciones IP.
  • DHCP — Asigna IPs automáticamente a dispositivos en la red.
  • ARP — Resuelve IPs a direcciones MAC en la red local.
  • ICMP — Mensajes de control (ping, traceroute).
# TCP Handshake de 3 vías Cliente ➔ SYN ➔ Servidor (inicio de conexión) Cliente ⬚ SYN-ACK ⬚ Servidor (confirmación) Cliente ➔ ACK ➔ Servidor (conexión establecida) # Ping (ICMP) — verificar conectividad ping 8.8.8.8 ping google.com # Traceroute — rastrear ruta de paquetes tracert google.com (Windows) traceroute google.com (Linux/Mac)

Direccionamiento IP y Subnetting

IPv4 usa 32 bits divididos en 4 octetos. El subnetting divide una red en subredes más pequeñas.

  • Clases de red: A (0-127), B (128-191), C (192-223).
  • IPs privadas: 10.x.x.x, 172.16-31.x.x, 192.168.x.x.
  • Máscara de red: separa la parte de red y de host.
  • Notación CIDR: 192.168.1.0/24 equivale a máscara 255.255.255.0.
  • /24 ➔ 256 IPs totales, 254 hosts, máscara 255.255.255.0
  • /16 ➔ 65536 IPs totales, 65534 hosts, máscara 255.255.0.0
  • /8 ➔ 16 millones de IPs, máscara 255.0.0.0
  • Broadcast: última IP de la subred (ej: 192.168.1.255 en /24).
  • Gateway: primera IP utilizable, conecta la red a Internet.
# Ejemplo: Red 192.168.1.0/24 Red: 192.168.1.0 Máscara: 255.255.255.0 Gateway: 192.168.1.1 Hosts: 192.168.1.1 - 192.168.1.254 (254 hosts) Broadcast: 192.168.1.255 # CIDR: /24 = 24 bits de red, 8 bits de host # 2^8 = 256 IPs totales - 2 (red+broadcast) = 254 hosts

Seguridad en Redes

Conceptos fundamentales de seguridad para proteger la infraestructura y los datos en tránsito.

  • Firewall — Filtra tráfico entrante/saliente según reglas. Puede ser de red o de host.
  • VPN (Virtual Private Network) — Crea un túnel cifrado sobre redes públicas.
  • TLS/SSL — Cifra las comunicaciones en la capa de transporte (HTTPS).
  • IDS / IPS — Detección/Prevención de intrusiones. IDS alerta, IPS bloquea.
  • Ataque DoS / DDoS — Inunda el servidor con solicitudes para dejarlo inaccesible.
  • Man-in-the-Middle (MITM) — El atacante intercepta la comunicación entre dos partes.
  • Phishing — Engaño para obtener credenciales mediante sitios/correos falsos.
  • VLAN — Segmenta redes lógicamente en el mismo switch físico.
  • NAT — Traduce IPs privadas a una IP pública para acceder a Internet.
  • Proxy — Servidor que actúa como intermediario entre un cliente y un servidor de destino para caché y control.
  • STP (Spanning Tree Protocol) — Evita bucles físicos en switches desactivando puertos redundantes de forma lógica.
  • Loopback IPv6 (::1) — Dirección especial de retorno local que representa al localhost en IPv6.
💡 Dato clave: HTTPS no garantiza que el sitio sea seguro o legítimo, solo que la comunicación está cifrada. Siempre verificar el dominio.
