Variables: var, let y const

ES6 introdujo let y const para superar las limitaciones de var.

  • var — Alcance de función, sufre hoisting, permite redeclaración. Evitar en código moderno.
  • let — Alcance de bloque, no se puede redeclarar en el mismo bloque, sí reasignar.
  • const — Alcance de bloque, no se puede reasignar. Obligatorio para referencias a objetos/arrays.
  • Hoisting: var se "sube" al inicio de la función con valor undefined. let/const también se elevan pero quedan en Temporal Dead Zone.
var x = 1; // Alcance de función let y = 2; // Alcance de bloque const z = 3; // No reasignable if (true) { let bloque = 'solo aquí'; // var x sigue accesible aquí } // bloque no existe fuera del if const obj = { nombre: 'Ana' }; obj.nombre = 'Luis'; // ✅ Mutar propiedades sí es válido // obj = {} ❌ No se puede reasignar la referencia
💡 Regla: Usa const por defecto. Solo cambia a let si necesitas reasignar. Nunca uses var.

Arrow Functions y Funciones

Las arrow functions tienen sintaxis compacta y no crean su propio contexto this.

  • Función tradicional: tiene su propio this, puede ser constructor, se eleva con hoisting.
  • Arrow function: hereda el this del contexto padre, no puede ser constructor, más concisa.
  • Retorno implícito: si el cuerpo es una sola expresión, se omiten llaves y return.
  • Parámetros por defecto: function suma(a, b = 0) asigna 0 si b no se pasa.
  • Rest params: ...args recoge todos los argumentos restantes en un array.
// Función tradicional function sumar(a, b) { return a + b; } // Arrow function equivalente const sumar = (a, b) => a + b; // Parámetro por defecto const saludar = (nombre = 'mundo') => `Hola, ${nombre}!`; // Rest params const total = (...nums) => nums.reduce((acc, n) => acc + n, 0); total(1, 2, 3, 4); // → 10

Arrays y Métodos Funcionales

Los métodos funcionales de arrays procesan colecciones de forma elegante y sin mutar el original.

  • map(fn) — Transforma cada elemento, devuelve nuevo array.
  • filter(fn) — Filtra elementos que cumplan condición.
  • reduce(fn, init) — Acumula un valor a partir de todos los elementos.
  • find(fn) — Devuelve el primer elemento que cumple la condición.
  • some(fn) / every(fn) — Booleano: alguno / todos cumplen condición.
  • flat() / flatMap(fn) — Aplana arrays anidados.
  • spread [...arr] — Copia o combina arrays.
  • destructuring — Extrae valores con [a, b] = arr.
const nums = [1, 2, 3, 4, 5]; nums.map(n => n * 2); // [2,4,6,8,10] nums.filter(n => n % 2 === 0); // [2,4] nums.reduce((acc, n) => acc + n, 0); // 15 // Desestructuración const [primero, segundo, ...resto] = nums; // primero=1, segundo=2, resto=[3,4,5] // Combinar arrays const todos = [...nums, ...[6, 7]]; // [1..7]

Promesas y Async/Await

JavaScript maneja operaciones asíncronas (red, timers) mediante Promises y la sintaxis async/await.

  • Promise — Objeto con 3 estados: pending, fulfilled, rejected.
  • .then(fn) — Ejecuta fn cuando la promesa se resuelve.
  • .catch(fn) — Captura errores o rechazos.
  • .finally(fn) — Siempre se ejecuta independientemente del resultado.
  • async function — Siempre retorna una promesa implícitamente.
  • await — Pausa la ejecución hasta resolver la promesa (solo dentro de async).
  • Promise.all([]) — Espera que TODAS las promesas resuelvan en paralelo.
  • Promise.race([]) — Resuelve con la primera promesa que termine.
// Con .then/.catch fetch('/api/datos') .then(res => res.json()) .then(data => console.log(data)) .catch(err => console.error(err)); // Con async/await (más legible) async function cargarDatos() { try { const res = await fetch('/api/datos'); const data = await res.json(); return data; } catch (err) { console.error('Error:', err); } }

Manipulación del DOM y Eventos

El DOM (Document Object Model) es la representación en árbol del HTML. JS puede modificarlo en tiempo real.

  • getElementById / querySelector — Seleccionar elementos.
  • querySelectorAll — Devuelve NodeList con todos los coincidentes.
  • textContent / innerHTML — Leer o escribir contenido (innerHTML permite HTML).
  • classList.add/remove/toggle — Gestionar clases CSS.
  • style.property — Cambiar estilos en línea directamente.
  • createElement / appendChild — Crear e insertar nodos.
  • addEventListener(evento, fn) — Escuchar eventos del usuario.
  • event.preventDefault() — Evitar comportamiento por defecto del navegador.
// Seleccionar y modificar const btn = document.querySelector('#mi-btn'); btn.textContent = 'Haz clic'; btn.classList.add('activo'); // Escuchar eventos btn.addEventListener('click', (e) => { e.preventDefault(); console.log('¡Clic detectado!'); }); // Crear e insertar elemento const li = document.createElement('li'); li.textContent = 'Nuevo ítem'; document.querySelector('ul').appendChild(li);

Módulos ES6 y LocalStorage

Los módulos permiten organizar el código en archivos reutilizables. LocalStorage persiste datos en el navegador.

  • export — Exporta una función, clase o variable de un módulo.
  • import — Importa desde otro módulo.
  • export default — Exportación principal del archivo.
  • localStorage.setItem(key, val) — Guarda dato (siempre string).
  • localStorage.getItem(key) — Lee dato guardado.
  • JSON.stringify / JSON.parse — Convierte objetos a/desde string para guardarlos.
// utils.js export const sumar = (a, b) => a + b; export default function saludar(n) { return `Hola ${n}`; } // main.js import saludar, { sumar } from './utils.js'; // LocalStorage const usuario = { nombre: 'Ana', rol: 'admin' }; localStorage.setItem('usuario', JSON.stringify(usuario)); const guardado = JSON.parse(localStorage.getItem('usuario'));
