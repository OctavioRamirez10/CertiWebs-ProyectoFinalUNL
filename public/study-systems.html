Arquitectura del Sistema Operativo

El SO actúa como intermediario entre el hardware y las aplicaciones de usuario. Sus componentes principales son:

  • Kernel (núcleo) — Núcleo del SO. Controla hardware, memoria, procesos y sistema de archivos. Tiene acceso privilegiado (modo kernel).
  • Modo usuario vs Modo kernel — Las apps corren en modo usuario con acceso restringido. El kernel corre con acceso total al hardware.
  • Kernel monolítico — Todo en un solo bloque (Linux, Unix). Más rápido pero si falla una parte, falla todo.
  • Microkernel — Solo lo esencial en modo privilegiado (QNX, Minix). Más estable y seguro, algo más lento.
  • Kernel híbrido — Combinación (Windows NT, macOS). Equilibrio entre rendimiento y modularidad.
  • Shell — Interfaz entre usuario y kernel. Puede ser gráfica (GUI) o de línea de comandos (CLI).
  • Llamadas al sistema (syscalls) — API del kernel: read(), write(), fork(), exec().
💡 Dato: Linux usa un kernel monolítico modular: tiene módulos (.ko) que se cargan/descargan en tiempo de ejecución, combinando velocidad y flexibilidad.

Gestión de Procesos y Planificación

Un proceso es un programa en ejecución con su propio espacio de memoria. Un hilo (thread) es una unidad de ejecución dentro de un proceso.

  • Estados de un proceso: New, Ready, Running, Waiting, Terminated.
  • PCB (Process Control Block) — Estructura del SO que almacena el estado del proceso (PID, registros, prioridad).
  • PID — Process ID, identificador único de cada proceso.
  • fork() — Crea una copia exacta del proceso actual (proceso hijo).
  • exec() — Reemplaza el proceso actual con un nuevo programa.
  • Algoritmos de planificación:
  • FCFS (First Come First Served) — Simple, puede causar convoy effect.
  • Round Robin — Quantum de tiempo fijo, equitativo para todos los procesos.
  • Prioridad — Procesos de mayor prioridad van primero, puede causar inanición.
  • SJF (Shortest Job First) — Mínimo tiempo promedio de espera.
  • Deadlock (Interbloqueo): Estado de bloqueo indefinido en el que dos o más procesos se esperan mutuamente para liberar recursos que cada uno posee.
# Ver procesos en Linux ps aux # Todos los procesos ps aux | grep nginx # Filtrar por nombre top # Monitor en tiempo real htop # Monitor interactivo (requiere instalación) kill -9 1234 # Terminar proceso con PID 1234 killall nginx # Terminar todos los procesos 'nginx' nice -n 10 cmd # Ejecutar con prioridad reducida

Gestión de Memoria

El SO administra la memoria RAM distribuyendo espacio entre procesos y el kernel.

  • Memoria virtual — Permite que procesos usen más memoria de la RAM física disponible, usando espacio en disco (swap).
  • Paginación — Divide la memoria en páginas de tamaño fijo. Elimina fragmentación externa.
  • Segmentación — Divide la memoria en segmentos lógicos (código, datos, pila).
  • Swap / Área de intercambio — Espacio en disco usado como extensión de RAM cuando esta se agota.
  • Thrashing — El SO pasa más tiempo haciendo swap que ejecutando procesos. Degradación severa del rendimiento.
  • Algoritmos de reemplazo de páginas: FIFO, LRU (Least Recently Used), Optimal.
  • Heap vs Stack — Stack: memoria automática de funciones. Heap: memoria dinámica (malloc/free).
  • Fragmentación externa: Existe suficiente memoria total libre para satisfacer un requerimiento, pero esta no se encuentra contigua (está dispersa en bloques pequeños).

Sistema de Archivos y Permisos Linux

Linux organiza todo en un árbol de directorios. Los permisos controlan quién puede leer, escribir o ejecutar.

  • / — Raíz del sistema de archivos.
  • /etc — Configuraciones del sistema y servicios.
  • /home — Directorios personales de usuarios.
  • /var — Datos variables: logs, bases de datos, correo.
  • /tmp — Archivos temporales, se borran al reiniciar.
  • /bin, /usr/bin — Ejecutables del sistema y usuarios.
  • Permisos: r (4) = lectura, w (2) = escritura, x (1) = ejecución.
  • chmod 755 = rwx(7) para owner, rx(5) para grupo y otros.
  • chown usuario:grupo archivo — Cambiar propietario.
  • Sistemas de archivos: Los nativos más comunes son ext4 para sistemas Linux y NTFS para Windows.
# Permisos: [tipo][owner][group][others] # -rwxr-xr-x = archivo, owner rwx, group r-x, others r-x chmod 755 script.sh # rwxr-xr-x chmod 644 config.txt # rw-r--r-- chmod +x script.sh # Agregar permiso de ejecución chown root:root /etc/nginx.conf # Tipos de archivos # - archivo regular # d directorio # l enlace simbólico # b dispositivo de bloque

Virtualización y Contenedores

La virtualización permite ejecutar múltiples SOs en un mismo hardware físico.

  • Máquina Virtual (VM) — Emula hardware completo. Aislamiento total. Más pesada. Ejemplos: VirtualBox, VMware, Hyper-V.
  • Hipervisor Tipo 1 — Corre directamente sobre el hardware (ESXi, Hyper-V en bare-metal). Mayor rendimiento.
  • Hipervisor Tipo 2 — Corre sobre un SO anfitrión (VirtualBox, VMware Workstation). Más fácil de usar.
  • Contenedor (Docker) — Comparte el kernel del host. Más liviano que una VM, arranca en segundos.
  • Imagen Docker — Plantilla de solo lectura para crear contenedores (Dockerfile).
  • Docker Compose — Define y ejecuta aplicaciones multi-contenedor con YAML.
  • Kubernetes — Orquestador de contenedores para entornos de producción a escala.
# Comandos Docker esenciales docker pull nginx # Descargar imagen docker run -p 80:80 nginx # Ejecutar contenedor docker ps # Contenedores activos docker images # Imágenes locales docker stop <id> # Detener contenedor docker rm <id> # Eliminar contenedor docker build -t mi-app . # Construir imagen desde Dockerfile
