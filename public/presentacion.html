Presentación y Banco de Respuestas

Documentación técnica de arquitectura y guía académica del sistema de exámenes de CertiWebs.

🎓 Proyecto CertiWebs

Sistema Educativo de Simulación de Exámenes y Certificación Técnica

Desarrollado para la Cátedra de Proyecto Final

Universidad Nacional del Litoral (UNL)

1. Resumen y Objetivos del Proyecto

CertiWebs es una plataforma diseñada para resolver la brecha de evaluación formativa en informática. Permite evaluar y certificar conocimientos en tiempo real, garantizando dinamismo y seguridad académica.

🎯 Misión

Ofrecer a los estudiantes herramientas interactivas de simulación con feedback inmediato e informes explicativos detallados.

💡 Autoevaluación

Permitir la práctica repetida mediante bancos de preguntas extensos, evitando la memorización repetitiva.

🔒 Validación

Generar certificados digitales en formato PDF firmados con códigos criptográficos verificables públicamente.

2. Arquitectura de Software (Stack)

Se estructuró una arquitectura robusta, ligera y veloz, libre de frameworks pesados de renderizado frontal, priorizando CSS vainilla y JavaScript modular nativo.

🎨 Frontend (Capa Cliente)

  • HTML5 Semántico para SEO y accesibilidad
  • CSS Vainilla moderno con variables nativas
  • JavaScript ES6 modular para lógica reactiva
  • Generación dinámica de certificados con jsPDF

⚙️ Backend (Capa Servidor)

  • Node.js + Express para APIs REST HTTP
  • Sesiones seguras con JSON Web Tokens (JWT)
  • Autenticación con hashing bcrypt
  • Servicio de emails asíncrono vía Nodemailer

🗄️ Base de Datos

  • SQLite3 para persistencia integrada ligera
  • Consultas indexadas de usuarios y certificados
  • Migrations seguras integradas al arranque
  • Configuración ágil con variables .env

3. Mecánica Interna del Examen

El núcleo pedagógico consta de simulaciones dinámicas configuradas con las mejores prácticas de experiencia de usuario (UX).

🎲 Selección Aleatoria

Se baraja la pila de 20 preguntas usando el algoritmo de Fisher-Yates simplificado y se seleccionan exactamente 5 para el intento actual.

shuffledPool.sort(() => 0.5 - Math.random())

🧭 Navegación Libre

Barra de círculos interactivos (Progress Dots). El alumno puede saltar a cualquier pregunta, ver cuáles están contestadas y cambiar respuestas libremente.

⏰ Alerta de Temporizador

Cronómetro regresivo. Al restar menos de 60 segundos, el diseño del temporizador añade la clase CSS `.warning` parpadeante en neón rojo.

4. Generación y Validación de Diplomas

CertiWebs no solo califica, sino que expide diplomas con validación formal para prevenir falsificaciones en el ámbito académico.

📄 Generación PDF Cliente

El PDF se ensambla directamente en el navegador del usuario utilizando jsPDF, reduciendo el consumo de recursos de CPU en el servidor en Render.

🔑 Código de Verificación

Cada certificado guardado almacena un UUID único asignado en la base de datos SQLite y asociado al ID del alumno y la materia aprobada.

🔍 Portal de Auditoría

Cualquier persona o empresa interesada puede verificar la autenticidad de un certificado ingresando el código único en el portal `/verificar.html`.

1. Resumen Ejecutivo del Proyecto

Este informe detalla los fundamentos de desarrollo de CertiWebs, una plataforma interactiva web dedicada a la enseñanza y acreditación de conocimientos tecnológicos fundamentales en el área informática. El sistema comprende guías de estudio interactivas, un simulador de exámenes dinámicos de selección aleatoria con temporizador de alerta, y un generador local de certificados PDF con códigos de validación criptográficos registrados en base de datos.

2. Estructura y Arquitectura del Sistema

Frontend (Capa de Presentación)

  • HTML5 Estructurado Semánticamente
  • Hojas de estilo CSS3 responsivas personalizadas (sin librerías)
  • Lógica de cliente modular nativa basada en Javascript ES6
  • Librería jsPDF en cliente para generación rápida de certificados

Backend y Persistencia

  • Servidor HTTP y API REST en Node.js con Framework Express
  • Gestión de usuarios y autenticación con tokens JWT (JSON Web Tokens)
  • Cifrado de claves de usuarios con bcrypt
  • Base de datos relacional integrada SQLite3

3. Lógica de los Exámenes y Feedback

Cada materia cuenta con un banco oficial de 20 preguntas especializadas. Para evitar la memorización mecánica, el servidor selecciona 5 preguntas al azar por intento. Al finalizar, el sistema provee al alumno la calificación final acompañada de una revisión minuciosa donde cada pregunta expone no solo la opción correcta sino una explicación teórica detallada de su validez.
